## Obsidian Local REST API — API Key Fix

**Issue:** Plugin generated its own API key on first load, ignoring the manually-written key in `data.json`. All protected endpoints returned `errorCode: 40101`.

**Root cause:** `data.json` contained key `0f0c04...` but the plugin was actually using key `efcd07...` (visible in Obsidian Settings).

**Fix:** Updated `.obsidian/plugins/obsidian-local-rest-api/data.json` with the real key:
```json
{
  "apiKey": "efcd07de6074823f8c295976763bee6599d68fd3caa61ad047c1c42f4523c91f",
  "port": 27124,
  "enableInsecureServer": true,
  "insecurePort": 27123
}
```

**Verified:**
- `GET /` → `"authenticated": true`
- `GET /vault/` → file listing on both HTTP (27123) and HTTPS (27124)
